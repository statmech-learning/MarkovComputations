What was achieved

A complete first-order topology test, not just a code scaffold

The original question was whether CRN topology matters for ICL after raw degrees of freedom are controlled. We implemented the required theory, ran matched fixed-edge experiments on Engaging, computed pre-training tree geometry, analyzed trained mechanisms, retrained extracted essential subgraphs and input masks, and passed strict artifact verification.

The answer is positive for this first-order setting. Raw edge count alone had essentially no explanatory power in the fixed-edge regime. Relative tree dimension, masked tree geometry, and especially post-training projection alignment explained substantial residual variation in novel-class ICL accuracy.

Scientific claim supported

Successful CRN ICL requires enough relative spanning-tree projection capacity to cover query-context comparison branches. Trained networks realize this through active tree ensembles and essential physical motifs.

Mathematical spine

Topology enters through rooted spanning trees

For a strongly connected first-order CRN, the steady state is governed by the matrix-tree theorem:

\[ \bar C_r(z)=\frac{\tau_r(z)}{\sum_s \tau_s(z)},\qquad \tau_r(z)=\sum_{T\in\mathcal T_r(G)}\prod_{e\in T} k_e(z). \]

With exponential encoding \(k_e(z)=\exp(b_e+K_e^\top z)\), each tree contributes a learned projection:

\[ \prod_{e\in T}k_e(z)= \exp\left(\beta_T+\Theta_T^\top z\right), \quad \Theta_T=\sum_{e\in T}K_e. \]

The key object is not the raw edge vector \(K_e\). It is the set of relative tree-sum projections \(K^\top(s_T-s_{T_0})\) that survive normalization.
